## 1. 认知升级：从"分工"到"协作理论"

### 1.1 前一个日志的局限

前一个日志 (`multi-lobster-coordination.md`) 将多龙虾协作主要建模为**项目管理问题**：
- 任务分解 → 角色分配 → 进度追踪
- 本质上是人类软件工程的管理范式

Enricher 的点评揭示了更深的层面：**这是一个多智能体协作 (Multi-Agent Collaboration, MAC) 问题**，应该用多智能体系统的理论框架来分析。

### 1.2 本质区别

| 维度 | 项目管理视角 (前一个日志) | 多智能体协作视角 (本日志) |
|------|------------------------|------------------------|
| 核心问题 | 怎么分活 | 怎么让群体智能涌现 |
| 优化目标 | 最短完成时间 | 最优决策质量 + 效率 |
| 协调方式 | 中央调度 (1号分配) | 涌现式协调 (机制设计) |
| 通信模型 | 文件共享 (被动) | 主动消息传递 + 共享状态 |
| 知识假设 | 每只龙虾能力相同 | 每只龙虾可能有不同的知识/视角 |
| 失败模式 | 单点故障 | 群体思维、级联失败、协调死锁 |

## 2. 多智能体协作的四个理论维度

### 2.1 维度一：通信拓扑 (Communication Topology)

智能体之间如何传递信息？

```
A. 全连接 (All-to-All)        B. 星型 (Star/Hub)
   🦞─🦞                        🦞
   │╲│╱│                       ╱│╲
   🦞─🦞─🦞                  🦞─🦞1号─🦞
   │╱│╲│                       ╲│╱
   🦞─🦞                        🦞
   
   通信成本 O(N²)              通信成本 O(N)
   信息完整度 最高              存在瓶颈
   
C. 分层 (Hierarchical)        D. 环形 (Ring)
      🦞1号                     🦞→🦞
      ╱  ╲                     ↑    ↓
   🦞组长 🦞组长                🦞←🦞
   ╱╲    ╱╲
  🦞 🦞  🦞 🦞
  
  通信成本 O(N log N)          通信成本 O(N)
  可扩展性 最强                 延迟 O(N)
```

**EvoAgent 当前**: 星型 (EA-001 调度器是 Hub)
**谢一凡论文**: 环形变体 (A→B→裁判→A 循环辩论)
**最优选择**: 取决于任务性质

> **关键思考**: 
> 对于 EvoAgent 的多路径探索，当前星型拓扑是否最优？
> 
> 星型的问题：路径之间完全隔离，无法互相启发。
> 环形的问题：通信延迟，但路径间可以"偷听"其他路径的发现。
> 
> **可能的改进**: EA-305 通信总线不应该只是"结果收集"，
> 而应该支持路径间的中间发现共享——类似于蚁群的信息素机制。

### 2.2 维度二：决策机制 (Decision Mechanism)

群体如何达成最终决定？

| 机制 | 描述 | 理论基础 | EvoAgent 对应 |
|------|------|---------|--------------|
| 多数投票 | 简单计数 | 孔多塞陪审团定理 | EA-004 |
| 加权投票 | 按历史准确率加权 | 专家系统 | EA-104 (部分) |
| 辩论+裁决 | 交互式论证 | 谢一凡论文 | EA-003 |
| 拍卖/市场 | 出价竞争 | 机制设计理论 | 未实现 |
| 贝叶斯聚合 | 概率融合 | 贝叶斯推理 | 未实现 |
| 共识协议 | 迭代收敛 | 分布式系统 (Paxos/Raft) | EA-009 (早停) |

> **关键思考**:
> 谢一凡论文证明了 辩论+裁决 > 简单投票 (92.86% vs 87.14%)。
> 但论文也发现了辩论的三个失败模式。
> 
> EvoAgent 目前用 多数投票优先 + LLM Judge 回退 (DD-003)。
> 这其实是一种"懒惰辩论"——只有当投票无共识时才启动辩论。
> 
> 更激进的方案：**总是辩论，投票只是辩论的输入之一**。
> 但这会增加成本。需要实验验证。

### 2.3 维度三：知识共享 (Knowledge Sharing)

智能体之间共享什么级别的信息？

```
Level 0: 只共享最终结果        ← EvoAgent 当前 (EA-003 只看最终答案)
Level 1: 共享结论 + 置信度     ← 可实现
Level 2: 共享推理过程          ← 谢一凡论文的辩论
Level 3: 共享中间发现          ← EA-305 通信总线的目标
Level 4: 共享工具调用结果      ← 可能的 EA-306 缓存层
Level 5: 共享内部状态/注意力   ← 理论上可行但成本极高
```

> **关键思考**:
> Level 0→Level 2 的提升是谢一凡论文的核心贡献 (辩论 = 共享推理过程)。
> Level 2→Level 4 是 EvoAgent EA-305/306 的目标。
> 
> 但 Level 4 有一个被忽视的价值：
> **如果路径 A 搜索了"巴菲特指数"并得到了结果，路径 B 不需要重复搜索。**
> 这不仅节省成本 (EA-306 缓存)，更重要的是**共享发现能改变其他路径的探索方向**。
> 
> 这就是蚁群算法中"信息素"的作用：
> 一只蚂蚁找到食物后留下信息素 → 其他蚂蚁倾向于沿信息素浓度高的路径走。
> 类比：一条路径找到关键证据 → 其他路径可以基于这个证据深入挖掘。

### 2.4 维度四：进化与适应 (Evolution & Adaptation)

系统如何从经验中改进？

```
      ┌─────────────────────────────────────────────┐
      │          进化时间尺度                          │
      │                                              │
      │  即时适应        跨任务进化       元进化       │
      │  (单次任务内)    (多任务积累)     (系统自改)   │
      │                                              │
      │  EA-009 早停     EA-101 战绩      EA-201 生成  │
      │  EA-012 重试     EA-104 选择      EA-202 淘汰  │
      │                  EA-108 经验      EA-203 仓库  │
      │                                              │
      │  ← 毫秒~分钟 → ← 小时~天 →  ← 天~周 →     │
      └─────────────────────────────────────────────┘
```

> **关键思考**:
> 多龙虾协作中也存在三个时间尺度的进化：
> - 即时：这次任务中龙虾2号发现 bug，立即通知龙虾3号避开
> - 跨任务：龙虾2号总是比龙虾3号更擅长写核心模块，下次直接分配
> - 元：发现角色分配本身不合理，调整组织架构
> 
> 这验证了 EvoAgent 三层架构的必要性。

## 3. 多龙虾协作的形式化模型

### 3.1 问题形式化

设有 N 只龙虾 L = {l₁, l₂, ..., lₙ}，任务集合 T = {t₁, t₂, ..., tₘ}

每只龙虾 lᵢ 在任务 tⱼ 上有：
- 能力值 cᵢⱼ ∈ [0, 1]
- 执行成本 eᵢⱼ (token/时间)
- 协调成本 coord(lᵢ, lₖ) 用于与 lₖ 协作

优化目标：
```
max Σⱼ quality(tⱼ)           # 最大化所有任务的完成质量
s.t. Σᵢⱼ eᵢⱼ·xᵢⱼ ≤ Budget   # 总成本约束
     Σⱼ xᵢⱼ ≤ Capacityᵢ     # 每只龙虾容量约束
     xᵢⱼ ∈ {0, 1}           # 分配变量
```

其中 quality(tⱼ) 不仅取决于执行者的能力，还取决于：
- 是否有其他龙虾 review (交叉验证)
- 是否与依赖任务的执行者有效同步 (协调质量)
- 是否存在多样性视角 (避免群体思维)

### 3.2 与 Multi-Armed Bandit 的关系

如果我们将"选择哪只龙虾做哪个任务"看作一个 MAB 问题：
- 每只龙虾 = 一个臂 (arm)
- 任务质量 = 回报 (reward)
- 不确定性 = 我们不知道每只龙虾在特定任务上的真实能力

这直接映射到 EA-104 的 Exploit/Explore 框架！

```python
# 策略选择 = 龙虾选择 = 多臂赌博机
def select_agent(task, agents, history):
    for a in agents:
        if history.count(a, task.type) < min_samples:
            return a  # Explore: 尝试未充分验证的组合
    
    # Exploit: 选择历史最优
    return max(agents, key=lambda a: history.win_rate(a, task.type))
```
