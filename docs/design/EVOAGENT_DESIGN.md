# EvoAgent 设计文档

> **项目代号**: EvoAgent  
> **基线项目**: MiroThinker v1.0  
> **核心理念**: 将进化搜索方法论应用于 Research Agent，通过多路径探索和策略进化超越单链路 ReAct 循环  
> **分支**: `evo-multi-path`  
> **创建日期**: 2026-03-13  
> **最后更新**: 2026-03-13

## 2. 功能清单与编号

### 2.1 第一层：多路径并行探索（Run-time）

| 编号 | 功能名称 | 描述 | 状态 | 优先级 |
|------|---------|------|------|--------|
| **EA-001** | 多路径调度器 | 对同一任务启动 N 条并行 Agent 路径，每条使用不同策略 | ✅ 已实现 | P0 |
| **EA-002** | 策略变体定义 | 定义可插拔的策略模板（breadth_first / depth_first / lateral_thinking） | ✅ 已实现 | P0 |
| **EA-003** | LLM 投票评选 | 多路径结果不一致时，用 LLM Judge 选出最优答案 | ✅ 已实现 | P0 |
| **EA-004** | 多数投票快速路径 | 多路径结果一致时，跳过 LLM Judge，直接采用多数答案 | ✅ 已实现 | P0 |
| **EA-005** | 独立工具管理器 | 每条路径拥有独立的 ToolManager 实例，避免状态冲突 | ✅ 已实现 | P0 |
| **EA-006** | 路径级日志隔离 | 每条路径生成独立的 TaskLog，支持单独审查和对比分析 | ✅ 已实现 | P1 |
| **EA-007** | 主控日志聚合 | 主调度器汇总所有路径结果、投票过程、最终选择到 master log | ✅ 已实现 | P1 |
| **EA-008** | 路径数动态配置 | 通过环境变量 `NUM_PATHS` 或配置文件控制并行路径数 | ✅ 已实现 | P1 |
| **EA-009** | 早停机制 | 当前 K 条路径达成共识时，取消剩余路径以节省成本 | ❌ 待开发 | P1 |
| **EA-010** | 路径预算分配 | 不同策略分配不同的 max_turns 预算（如广搜少、深搜多） | ❌ 待开发 | P2 |
| **EA-011** | 异步流式输出 | 各路径的中间思考过程实时流式输出到前端 | ❌ 待开发 | P2 |
| **EA-012** | 路径失败重试 | 路径因 API 错误等原因失败时，自动用备选策略重启 | ❌ 待开发 | P2 |

### 2.2 第二层：跨任务策略进化（Cross-task）

| 编号 | 功能名称 | 描述 | 状态 | 优先级 |
|------|---------|------|------|--------|
| **EA-101** | 策略-结果记录器 | 每次任务完成后，记录「策略 × 任务类型 → 分数」映射 | ❌ 待开发 | P1 |
| **EA-102** | 策略效果数据库 | 持久化存储策略在不同任务类型上的历史表现（JSONL格式） | ❌ 待开发 | P1 |
| **EA-103** | 任务分类器 | 自动判断任务类型（知识问答 / 多跳推理 / 计算 / 信息检索） | ❌ 待开发 | P1 |
| **EA-104** | 自适应策略选择 | 根据任务类型，从历史数据中选择表现最佳的 K 个策略 | ❌ 待开发 | P1 |
| **EA-105** | 策略参数进化 | 用进化算法优化策略参数（搜索深度、工具权重、prompt模板变量） | ❌ 待开发 | P2 |

## 3. 策略定义规范

每个策略是一个 Python dict，结构如下：

```python
{
    "name": "strategy_unique_name",       # 策略唯一标识
    "description": "Human-readable description",  # 策略描述
    "prompt_suffix": "...",               # 注入到 system prompt 末尾的指导文本
    "params": {                           # 可选：策略参数（供 EA-105 进化优化）
        "max_turns": 600,                 #   最大轮次
        "search_breadth": 3,              #   每轮搜索宽度
        "tool_preference_order": [...],   #   工具优先级
        "verification_rounds": 1,         #   自我验证次数
    }
}
```

### 3.1 预定义策略

| 策略名 | 编号 | 核心思路 |
|--------|------|---------|
| `breadth_first` | STR-01 | 先广泛搜索多个来源，再交叉验证 |
| `depth_first` | STR-02 | 找到一个权威来源后深入挖掘 |
| `lateral_thinking` | STR-03 | 从侧面角度切入，利用代码计算辅助 |
| `verification_heavy` | STR-04 | 找到答案后用多种方式交叉验证 (**待实现**) |
| `tool_specialist` | STR-05 | 优先使用代码执行工具而非搜索 (**待实现**) |

## 4. 数据流

```
输入任务
    │
    ▼
┌─────────────────────┐
│ EA-CTL: 调度中心     │
│                     │
│ 1. 接收任务          │
│ 2. 任务分类 (EA-103) │
│ 3. 策略选择 (EA-104) │
│ 4. 分配路径 (EA-001) │
└──────────┬──────────┘
           │
     ┌─────┼─────┐
     │     │     │
     ▼     ▼     ▼
   Path0 Path1 Path2    ← 并行执行 (EA-005 独立 ToolManager)
     │     │     │
     ▼     ▼     ▼
   Ans0  Ans1  Ans2     ← 各自产出答案
     │     │     │
     └─────┼─────┘
           │
     ┌─────▼─────┐
     │ EA-VOTE   │
     │           │
     │ 多数投票   │ ← EA-004
     │    或      │
     │ LLM Judge │ ← EA-003
     └─────┬─────┘
           │
     ┌─────▼─────┐
     │ 最终答案   │
     └─────┬─────┘
           │
     ┌─────▼──────────┐
     │ EA-101: 记录     │ ← 策略 × 任务类型 → 结果
     │ EA-102: 持久化   │
     └────────────────┘
```

## 6. 开发路线图

### Phase 1: 基础多路径（当前）✅
- EA-001 ~ EA-008, EA-301 ~ EA-303
- 目标：验证多路径探索比单路径更优

### Phase 2: 健壮性 + 测试
- EA-009 (早停), EA-010 (预算分配), EA-012 (失败重试)
- EA-304 (成本追踪)
- EA-401 ~ EA-404 (测试套件)
- 目标：生产级稳定性

### Phase 3: 策略进化
- EA-101 ~ EA-108
- EA-405 ~ EA-407 (基准对比)
- 目标：策略随使用自动优化

### Phase 4: 元进化（实验性）
- EA-201 ~ EA-203
- 目标：探索自我进化的上限

## 8. 设计决策记录

| 编号 | 决策 | 理由 | 日期 |
|------|------|------|------|
| DD-001 | 策略通过 prompt suffix 注入而非修改代码逻辑 | 最小侵入性，不改变原版 Orchestrator | 2026-03-13 |
| DD-002 | 每条路径使用独立 ToolManager | MCP 连接有状态，共享会导致冲突 | 2026-03-13 |
| DD-003 | 投票采用"多数优先 + LLM Judge 兜底" | 多数投票零成本，LLM Judge 仅在分歧时启用 | 2026-03-13 |
| DD-004 | 用 DuckDuckGo 替代 Serper | Serper key 无效，DuckDuckGo 免费且足够 | 2026-03-13 |
| DD-005 | 用本地 subprocess 替代 E2B | 无 E2B key，本地执行对测试足够 | 2026-03-13 |

## 9. 风险与缓解

| 风险 | 影响 | 概率 | 缓解措施 |
|------|------|------|---------|
| Token 成本 N 倍增长 | 高 | 高 | EA-009 早停、EA-306 缓存、EA-010 预算分配 |
| 并行路径竞争 API 限速 | 中 | 中 | 错开启动时间、实现退避机制 |
| 策略进化需要大量历史数据 | 中 | 中 | Phase 2 先积累数据，Phase 3 再启动进化 |
| LLM Judge 选错答案 | 中 | 低 | 多维评分（置信度 + 证据数量 + 来源可靠性） |

## 10. 术语表

| 术语 | 定义 |
|------|------|
| **Path（路径）** | 一次独立的 Agent ReAct 循环执行 |
| **Strategy（策略）** | 指导 Agent 搜索行为的 prompt 模板 + 参数 |
| **Vote（投票）** | 多路径结果汇聚后的最优选择过程 |
| **Judge（裁判）** | 评选最优答案的 LLM 调用 |
| **Turn（轮次）** | ReAct 循环中 LLM 调用 + 工具执行的一个周期 |
| **Population（种群）** | 当前活跃的策略集合 |
| **Fitness（适应度）** | 策略在特定任务类型上的历史正确率 |
